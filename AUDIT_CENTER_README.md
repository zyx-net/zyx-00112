# 导入替换审计中心 - 功能说明与使用指南

## 概述

导入替换审计中心是一个完整的审计追踪模块，用于记录、查询和验证场景包导入替换操作的全流程。该模块提供了详细的操作留痕、资源变更追踪和一致性验证功能。

## 核心功能

### 1. 导入批次管理
- **批次创建**: 每次导入操作自动创建唯一批次号
- **批次查询**: 支持分页查询所有导入批次
- **批次详情**: 查看批次的完整信息，包括操作者、时间、类型等

### 2. 快照版本链追踪
- **版本链记录**: 记录每个快照的版本号和创建时间
- **替换追踪**: 记录快照被替换的时间、原因和批次信息
- **关联执行**: 显示快照关联的执行记录和状态

### 3. 被替换快照详情
- **原始快照ID**: 被替换的快照标识
- **原始场景信息**: 快照所属场景名称
- **执行关联**: 快照关联的执行记录及状态
- **替换决策**: 导致替换的冲突决策和操作者
- **数据保留**: 保存原始快照数据用于审计

### 4. 回滚资源变更追踪
- **变更类型**: 区分removed(删除)和restored(恢复)操作
- **资源类型**: 场景、执行、快照、注入、API版本等
- **状态快照**: 记录变更前后的完整状态
- **关联恢复**: 记录恢复的关联关系，如快照与执行的映射

### 5. 重启复查流程
#### 模拟检查
- **目的**: 在不进行真实重启的情况下验证数据一致性
- **检查项**:
  - 场景是否存在
  - 执行记录完整性
  - 快照与执行的关联有效性
  - 数据完整性校验

#### 真实重启验证
- **目的**: 执行实际的数据一致性检查
- **验证内容**:
  - 执行-快照配对验证
  - 关联有效性检查
  - 操作者记录
  - 验证时间戳

### 6. 综合审计报告
- **批次信息**: 批次号、操作者、时间等
- **汇总统计**: 各类型资源变更数量
- **详细记录**: 完整的快照链、替换详情、回滚变更
- **问题清单**: 错误和警告列表
- **导出功能**: 支持生成JSON格式报告

## 数据库表结构

### import_audit_batches
导入批次主表，记录批次级别信息。

### snapshot_version_chain
快照版本链，记录快照的版本历史和替换关系。

### replaced_snapshot_details
被替换快照详情，记录被替换快照的完整信息。

### rollback_resource_changes
回滚资源变更，记录回滚操作的具体变更。

### restart_review_records
重启复查记录，区分模拟和真实重启验证。

## API 接口

### 批次管理
- `POST /api/audit-center/batches` - 创建批次
- `GET /api/audit-center/batches` - 查询批次列表
- `GET /api/audit-center/batches/:batchId` - 查询批次详情
- `GET /api/audit-center/batches/:batchId/report` - 生成审计报告

### 快照版本链
- `GET /api/audit-center/snapshot-versions/:scenarioId` - 查询场景的快照版本链

### 替换快照详情
- `GET /api/audit-center/replaced-snapshots/:batchId` - 查询批次的替换快照详情

### 回滚变更
- `GET /api/audit-center/rollback-changes/:batchId` - 查询批次的回滚变更详情

### 重启复查
- `GET /api/audit-center/restart-reviews/:batchId` - 查询批次的重启复查记录
- `POST /api/audit-center/restart-reviews/:batchId/simulation/:scenarioId` - 执行模拟检查
- `POST /api/audit-center/restart-reviews/:batchId/real-restart/:scenarioId` - 执行真实重启验证

## 使用场景

### 场景1: 导入替换并审计
1. 执行导入替换操作
2. 在审计中心查看批次列表
3. 点击批次查看替换快照详情
4. 执行模拟检查验证一致性
5. 生成审计报告存档

### 场景2: 回滚追踪
1. 执行回滚操作
2. 在审计中心查看回滚批次
3. 查看资源变更详情
4. 验证恢复的关联关系
5. 确认恢复的数据完整性

### 场景3: 重启后验证
1. 服务器重启后
2. 查询相关批次
3. 执行模拟检查初步验证
4. 执行真实重启验证确认
5. 查看验证结果和错误

## 注意事项

1. **数据保留**: 审计数据应长期保留用于合规审计
2. **性能考虑**: 大量审计记录时可使用分页和索引优化
3. **权限控制**: 生产环境应添加权限验证
4. **报告导出**: 大数据量时注意导出文件的存储空间

## 故障排查

### 批次列表为空
- 检查数据库连接
- 查看服务器日志确认审计记录是否创建

### 模拟检查失败
- 确认场景存在
- 检查执行和快照关联

### 真实重启验证失败
- 查看错误信息中的具体问题
- 检查数据库一致性
